# Update Nix Presets Documentation

This workflow regenerates the documentation (`README.md`) for the `nix-presets` sub-repository based on the current state of the codebase, and automatically commits the changes.

## Step 1: Analyze Firefox Configuration
Read the following files to understand the current Firefox setup:
- `nix-presets/firefox.nix`
- `nix-presets/firefox-settings.nix`

From these files, generate a Markdown matrix (table) showing which extensions are enabled across the different profiles (Standard, Laboratory, Vault, etc.). Use ✅ and ❌ to indicate presence.

## Step 2: Analyze Container Architecture
Read the directory `nix-presets/containers/` to identify which system containers are currently defined in the infrastructure.

## Step 3: Generate and Write README
// turbo
Overwrite the file at `nix-presets/README.md` with the new documentation.

**CRITICAL:** The very first line of the file MUST be:
`<!-- ⚠️ DO NOT EDIT THIS FILE DIRECTLY. Generated by AI via /presets_update_docs ⚠️ -->`

The `README.md` must contain:
1. **Title & Overview**: Brief description of the Nix Presets repository.
2. **Firefox Extension Matrix**: The table generated in Step 1.
3. **Active Containers**: A simple list of the containers discovered in Step 2.
4. **Usage**: A brief code block on how to import this flake (you can copy this from the original README).

## Step 4: Commit Changes to Sub-Repo
// turbo
Run the following terminal commands to commit the changes specifically to the `nix-presets` sub-repository. 
```bash
cd nix-presets
git add README.md
git commit -m "docs(auto): update README and firefox matrix via workflow"
```

Provide a brief summary of the changes made to the user after completion.
